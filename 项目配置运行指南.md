## 三、详细配置步骤

### 3.1 创建数据库

```sql
-- 1. 创建数据库
CREATE DATABASE IF NOT EXISTS web_test 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 2. 导入表结构（9张表）
--    article_category    文章分类
--    blog_details        博客详情
--    code_library        代码库
--    file_download       文件下载
--    image_upload        图片上传
--    message_area        留言区
--    release_feature     版本特性
--    user_info           用户信息
--    visit_total         访问统计

-- 3. 导入初始数据
--    文章分类 x1, 博客 x1, 代码 x1, 版本特性 x1, 用户 x3, 访问统计 x1
```

**SQL脚本路径：**
- 建表：`docs/数据库部署/第1步 - 创建数据库/*.sql`
- 初始数据：`docs/数据库部署/第2步 - 初始化数据.sql`

> **注意：** MySQL 8.0 默认使用 `caching_sha2_password` 认证插件，但项目使用的旧版 `mysql-connector-java-5.1.44` 不支持该插件。需将用户认证方式改为 `mysql_native_password`。

### 3.2 创建数据库用户

```sql
CREATE USER IF NOT EXISTS 'web_user'@'localhost' 
  IDENTIFIED WITH mysql_native_password BY '123456';

GRANT ALL PRIVILEGES ON web_test.* TO 'web_user'@'localhost';
FLUSH PRIVILEGES;
```

### 3.3 配置文件

项目配置文件位于 `web-pc/config/` 目录，部署时需放在 webapps 的上级目录（即 `webapps/config/`）。

#### 3.3.1 数据库连接配置

**文件：** `config/mysql_connect_info.properties`

```properties
# 数据库URL
url_psql=jdbc:mysql://127.0.0.1:3306/

# Linux服务器运行时使用的数据库名
dbname_official=web

# Windows本地运行时使用的数据库名
dbname_test=web_test

# 数据库用户名
username_webuser=web_user

# 数据库密码
password_webuser=123456

# 数据库驱动
driver_class=com.mysql.jdbc.Driver
```

#### 3.3.2 系统配置

**文件：** `config/config.properties`

```properties
# 发送邮箱（站内信功能）
email.from.user=your_email@qq.com
email.from.password=your_password
# 接收邮箱
email.to.user=your_email@qq.com
```

#### 3.3.3 日志配置

**文件：** `config/log4j2.xml`（使用模板即可，无需修改）

### 3.4 编译后端代码

```bash
# 设置变量
WEB_CORE="E:/document/chorm/web-master/web-master/web-core"

# 编译（所有依赖JAR在libs目录下）
CLASSPATH=$(find "$WEB_CORE/libs" -name "*.jar" | tr '\n' ';')
find "$WEB_CORE/src" -name "*.java" > /tmp/java_sources.txt
javac -encoding UTF-8 -cp "$CLASSPATH" -d "$WEB_CORE/build/classes" @/tmp/java_sources.txt

# 打包JAR
cd "$WEB_CORE/build/classes"
jar cfm web-core.jar "$WEB_CORE/src/META-INF/MANIFEST.MF" .

# 复制到web-pc的lib目录
cp web-core.jar "E:/document/chorm/web-master/web-master/web-pc/WebContent/WEB-INF/lib/"
```

### 3.5 安装配置 Tomcat

```bash
# 1. 下载 Tomcat 9.0.118
curl -L -o /tmp/apache-tomcat-9.zip \
  "https://mirrors.aliyun.com/apache/tomcat/tomcat-9/v9.0.118/bin/apache-tomcat-9.0.118-windows-x64.zip"

# 2. 解压
unzip /tmp/apache-tomcat-9.zip -d E:/tomcat/
```

### 3.6 部署项目

```bash
TOMCAT="E:/tomcat/apache-tomcat-9.0.118"

# 1. 部署Web应用（作为ROOT应用，去掉项目名称）
rm -rf "$TOMCAT/webapps/ROOT"
cp -r "web-pc/WebContent" "$TOMCAT/webapps/ROOT"

# 2. 复制配置文件到webapps/config/（与ROOT同级）
mkdir -p "$TOMCAT/webapps/config"
cp web-pc/config/mysql_connect_info.properties "$TOMCAT/webapps/config/"
cp web-pc/config/config.properties "$TOMCAT/webapps/config/"
cp web-pc/config/log4j2.xml "$TOMCAT/webapps/config/"

# 3. 移除自带的servlet-api.jar（避免与Tomcat容器冲突）
rm -f "$TOMCAT/webapps/ROOT/WEB-INF/lib/servlet-api.jar"
```

### 3.7 部署目录结构

```
E:/tomcat/apache-tomcat-9.0.118/
└── webapps/
    ├── ROOT/                       # Web应用（原WebContent）
    │   ├── index.jsp               # 首页
    │   ├── css/ js/ module/        # 前端资源
    │   ├── images/                 # 图片
    │   ├── personal_center/        # 个人中心页面
    │   └── WEB-INF/
    │       ├── web.xml
    │       ├── lib/                # 依赖JAR（含web-core.jar）
    │       └── classes/            # 编译类（如有）
    ├── config/                     # 配置文件（与ROOT同级，由WEB_ROOT_PATH定位）
    │   ├── mysql_connect_info.properties
    │   ├── config.properties
    │   └── log4j2.xml
    ├── logs/                       # 应用日志（自动创建）
    ├── download/                   # 文件下载目录（自动创建）
    └── images/                     # 图片上传目录（自动创建）
```

> **路径说明：** `WebInitConfigListener` 启动时获取 `servletContext.getRealPath("/")`（即ROOT目录），其父目录（webapps/）作为 `WEB_ROOT_PATH`，配置文件、日志、上传文件均相对于此路径。

## 七、常见问题

### 7.1 数据库认证插件错误

```
java.sql.SQLException: Unable to load authentication plugin 'caching_sha2_password'
```

**解决：** MySQL 8.0 默认使用 `caching_sha2_password`，旧版JDBC驱动不支持。

```sql
ALTER USER 'web_user'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456';
FLUSH PRIVILEGES;
```

### 7.2 JDK版本兼容性

项目使用JDK 23.0.1，Tomcat 9.0.118通过 `--add-opens` 参数兼容运行。无需额外配置。

### 7.3 端口冲突

如8080端口被占用，可修改Tomcat配置：

```xml
<!-- E:/tomcat/apache-tomcat-9.0.118/conf/server.xml -->
<Connector port="8080" protocol="HTTP/1.1" .../>
<!-- 将8080改为其他端口，如8081 -->
```

### 7.4 配置文件未找到

确保配置文件位于 `webapps/config/` 目录（与ROOT应用同级），文件名分别是：
- `mysql_connect_info.properties`
- `config.properties`
- `log4j2.xml`
